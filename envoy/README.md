## Поток данных

### 1. /me
Возвращает пользователю его username и telegram_id. Команда облегчает получение telegram_id
для выдачи админских прав.

### 2. /forecast
Позволяет заполнить запрос для формирования прогноза по активу, который будет отправлен в 
postman. JWT токены хранятся прямо в памяти бота. Схема подготовки токенов перед запросом
в postman выглядит следующим образом:
1. Пробуем запрос с access_token
2. Если 200–299 → ОК
3. Если 401 / 403: 
   1. Если есть refresh_token:

      1. если refresh ОК → сохраняем новый access → повторяем запрос
        
      2. если refresh не ОК → делаем auth

   2. Если refresh_token нет → auth

После получения валидных токенов делается запрос в postman. Если тот возвращает статус 200,
то мы сразу возвращаем пользователю прогноз, иначе ждем, пока он придет из kafka.
